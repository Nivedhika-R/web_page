Project Overview

XR-Care is an AR application that combines Unity's AR Foundation with WebRTC communication and a web-based annotation system. The project enables real-time AR camera feed sharing, annotation creation through a web interface, and synchronized display of annotations in AR space across multiple devices.

Complete Setup Guide

Phase 1: Unity 6 Installation & Configuration

1

Install Unity Hub and Unity 6

First, download and install Unity Hub from unity.com. Then install Unity 6 (Latest LTS) with the following modules:

  • Android Build Support
  • Android SDK & NDK Tools
  • OpenJDK
For Complete Beginners: Unity Hub is required to manage Unity installations and projects. Make sure you create a Unity account during the installation process.
2

Install Required Packages

Open Package Manager (Window → Package Manager) and install:

  • AR Foundation (latest version)
  • ARCore XR Plugin
  • XR Plugin Management
3

Configure XR Plugin Management

Navigate to Edit → Project Settings → XR Plug-in Management

  • Switch to Android platform tab
  • Enable ARCore provider
  • Ensure AR Foundation is properly configured

Phase 2: Project Setup

4

Install Git and Clone the Repository

First install Git from git-scm.com if not already installed. Then clone the repository:

git clone https://github.com/Nivedhika-R/xrai.git cd xrai

Open the project in Unity Hub by clicking "Add" and selecting the cloned folder.

Alternative: If you don't want to use Git, you can download the project as a ZIP file from GitHub and extract it.
5

Switch to Android Platform

Switch build platform to Android:

File → Build Settings → Android → Switch Platform
6

Configure Player Settings

Navigate to Edit → Project Settings → Player and configure:

  • Company Name: Your company name
  • Product Name: XR-Care
  • Package Name: com.yourcompany.xrcare
  • Minimum API Level: API Level 24 (Android 7.0)
  • Target API Level: API Level 33
  • Scripting Backend: IL2CPP
  • Target Architectures: ARM64 ✓
  • Internet Access: Require
  • Auto Graphics API: Uncheck this option
  • Graphics APIs: Remove Vulkan (keep only OpenGLES3)
Important: Uncheck Auto Graphics API and remove Vulkan to ensure compatibility with AR Foundation and ARCore.

Phase 3: Python Server Setup

7

Install Python and Navigate to Server Directory

First, install Python 3.8 or higher from python.org if not already installed. Make sure to check "Add Python to PATH" during installation.

Then navigate to the server directory:

cd Server~/
Verify Python Installation: Run python --version in command prompt/terminal to confirm Python is installed correctly.
8

Create SSL Certificate

Generate a self-signed certificate for HTTPS:

openssl req -new -x509 -keyout server.pem -out server.pem -days 365 -nodes
Note: Install OpenSSL if not available on your system.
9

Install Python Dependencies

pip install -r requirements.txt
10

Find Your IP Address and Start the Server

First, find your computer's IP address:

Windows: ipconfig Mac/Linux: ifconfig or use: hostname -I

Look for your local IP address (usually starts with 192.168.x.x or 10.0.x.x)

Then start the server with your IP address:

python server_annotation.py --ip <YOUR_IP_ADDRESS> --port 8000 --pem server.pem

Example: python server_annotation.py --ip 192.168.1.100 --port 8000 --pem server.pem

11

Launch Preview Application

In a separate terminal:

python preview_annotation.py --backend <YOUR_IP_ADDRESS> --bport 8000 --https

Phase 4: Unity Configuration & Deployment

12

Open XaiRScene

Navigate to the Scenes folder and open XaiRScene.unity.

13

Configure Network Settings

In the XAIRClient GameObject:

  • Set IP Address field to your server IP
  • Set Port to 8000
14

Enable Developer Mode on Android Device

Complete beginner setup for Android device:

  1. Go to Settings → About Phone
  2. Tap Build Number 7 times to enable Developer Options
  3. Go to Settings → Developer Options
  4. Enable USB Debugging
  5. Enable Stay Awake (optional but helpful)

Connect device via USB cable and verify connection:

adb devices

You should see your device listed. If not, install ADB drivers for your device.

ARCore Compatibility: Verify your device supports ARCore at developers.google.com/ar/devices
15

Build and Run

In Unity:

  • Go to File → Build Settings
  • Ensure XaiRScene is added to scenes
  • Select your Android device
  • Click Build and Run

File Structure & Descriptions

Unity Scripts

AcceptAnyCertificate.cs
This solves a common development problem with HTTPS certificates.
AndroidCameraManager.cs
Handles the phone camera for AR on Android devices. This implements our camera interface for Android AR using Unity's ARFoundation.
ARPlaceCube.cs
This is the heart of the AR system. It solves the hardest problem: converting 2D screen coordinates to 3D world positions.
ConcurrentWebRequestManager.cs
Solves a specific problem: some servers can't handle multiple requests at once.
ICameraDeviceManager.cs
This is an interface that defines what camera systems should be able to do. This pattern lets us swap between Android AR cameras and regular webcams without changing other code.
MediaManager.cs
Coordinates camera and microphone. It uses the Singleton pattern so there's only one and is your 'media coordinator'.
MicrophoneManager.cs
Handles audio recording with Android permissions.
ServerCommunication.cs
Manages all server interactions. It handles two completely different types of communication: WebRTC signaling for real-time connections and annotation polling for AR data.
Singleton.cs
Provides generic singleton pattern for MonoBehaviours to ensure only one copy of important managers exists.
WebCamManager.cs
Handles regular webcam on computers, implementing the same interface as the Android camera.
WebRTCController.cs
Manages direct connections between devices using WebRTC. WebRTC is complex because it's a state machine with many steps to work through firewalls and different network conditions.
XAIRClient.cs
This is the main controller that orchestrates everything. It's the conductor of the orchestra, coordinating all other systems.

Python Server Scripts

preview_annotation.py
Creates a web page for viewing AR camera feed and creating annotations using Gradio.
server_annotation.py
This is the backend server that coordinates Unity, WebRTC, and web interface. It's the central hub managing all communication.

System Data Flow

Complete Data Flow:

  1. Unity's AndroidCameraManager captures AR camera frames with 3D position data
  2. XAIRClient sends images to Python server via ServerCommunication
  3. server_annotation.py stores images and makes them available to web interface
  4. preview_annotation.py fetches images and lets users draw annotations
  5. Web annotations go back to server_annotation.py with coordinate data
  6. Unity polls server for new annotations via ServerCommunication
  7. ARPlaceCube converts 2D coordinates to 3D positions using pose matrices
  8. 3D annotation objects appear in AR space

WebRTC Connection Flow (runs in parallel):

  1. Unity logs into server, gets client ID
  2. WebRTCController creates connection offer or waits for offers
  3. server_annotation.py facilitates SDP and ICE candidate exchange
  4. Direct WebRTC connection established for real-time communication
  5. Data channel used for sending messages between Unity instances

Troubleshooting Guide

Unity Setup Issues

Unity Hub won't install Unity 6
Solution: Ensure you have sufficient disk space (at least 8GB free). Try running Unity Hub as administrator. If still failing, download Unity 6 directly from Unity Archive and install manually.
AR Foundation packages not installing
Solution: Make sure you're connected to the internet. Try refreshing Package Manager (Window → Package Manager → Refresh). If packages show errors, switch to "In Project" view and remove, then reinstall.
"ARCore provider not available" error
Solution: Ensure you've switched to Android platform first (File → Build Settings → Android → Switch Platform). Then go to Project Settings → XR Plug-in Management → Android tab and enable ARCore.
Build fails with "Vulkan" or graphics API errors
Solution: Go to Project Settings → Player → Android → Graphics APIs. Uncheck "Auto Graphics API" and remove Vulkan from the list. Keep only OpenGLES3. This is crucial for AR Foundation compatibility.

Python Server Issues

"Python not recognized" command error
Solution: Python is not in your system PATH. Reinstall Python from python.org and check "Add Python to PATH" during installation. Alternatively, use full path: C:\Python39\python.exe (adjust version/path as needed).
"pip install -r requirements.txt" fails
Solution: Try upgrading pip first: python -m pip install --upgrade pip. If still failing, install packages individually. For permission errors on Windows, run command prompt as administrator.
"openssl not recognized" error
Solution: Install OpenSSL from slproweb.com/products/Win32OpenSSL.html (Windows) or use brew install openssl (Mac). For quick testing, you can skip SSL and modify server.py to use HTTP instead of HTTPS.
Server starts but Unity can't connect
Solution: Check firewall settings - allow Python through Windows Firewall. Ensure both PC and Android device are on same WiFi network. Verify IP address is correct using ipconfig. Try disabling antivirus temporarily.

Android Device Issues

"adb devices" shows no devices
Solution: Install proper USB drivers for your device from manufacturer's website. Try different USB cable. Enable "File Transfer" mode on Android when connected. Restart ADB: adb kill-server then adb start-server.
Device shows "unauthorized" in adb
Solution: Look for USB debugging authorization popup on your Android device and tap "Always allow from this computer" then "OK". If no popup appears, revoke USB debugging authorizations in Developer Options and reconnect.
"Device doesn't support ARCore" error
Solution: Check your device compatibility at developers.google.com/ar/devices. If supported but still failing, install ARCore app from Google Play Store manually. Some devices need ARCore to be updated separately.
App installs but crashes immediately
Solution: Check Android logs: adb logcat. Common causes: missing permissions in AndroidManifest.xml, incompatible architecture (ensure ARM64 is selected), or missing ARCore installation. Try building in Development mode first.

Network and Communication Issues

Can't find IP address
Solution: Use these commands - Windows: ipconfig | findstr IPv4, Mac: ifconfig | grep inet, or visit whatismyipaddress.com for external IP. Your local IP usually starts with 192.168.x.x or 10.0.x.x.
WebRTC connection fails
Solution: Ensure both devices are on same network. Check router settings - some routers block peer-to-peer connections. Try using mobile hotspot for testing. Verify server is running and accepting connections.
Annotation system not working
Solution: Check if preview.py is running and accessible via web browser. Verify Unity is sending images to server (check server console output). Ensure coordinate conversion is working - test with simple annotations first.

General Performance Issues

AR tracking is jittery or unstable
Solution: Ensure good lighting conditions - AR tracking needs sufficient light. Clean camera lens. Move device slowly during initialization. Some devices perform better in landscape orientation.
App runs very slowly
Solution: Lower camera resolution in AndroidCameraManager settings. Reduce annotation polling frequency. Close other apps running on Android device. Use Release build instead of Debug build for better performance.

Quick Diagnostic Commands

Use these commands to diagnose issues:
• Check Unity Console for errors (Window → General → Console)
• Android logs: adb logcat | grep Unity
• Test Python server: Open http://YOUR_IP:8000 in browser
• Network test: ping YOUR_IP_ADDRESS
• Check ADB connection: adb devices -l
• Python version: python --version
• Unity version: Help → About Unity
